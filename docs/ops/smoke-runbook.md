# Smoke Runbook — 核心功能验证

> HD-015 | 核心冒烟测试手册，供开发人员在每次部署后按步骤验证。

## 前置条件

- agent-api 运行在 `http://localhost:3002`
- web 运行在 `http://localhost:3000`
- 浏览器：Chrome / Edge 最新版

---

## 1. 首页（Homepage）

| # | 步骤 | 预期结果 |
|---|------|----------|
| 1.1 | 打开 `http://localhost:3000` | 页面标题显示 "AI Health Advisor" |
| 1.2 | 观察晨报卡片 | 显示 "今日简报" 标题和摘要内容 |
| 1.3 | 观察 "智能洞察" 区域 | 显示彩色标签（微贴士 pills） |
| 1.4 | 观察 "历史趋势" 区域 | 4 个趋势卡片（HRV / 睡眠 / 步数 / 压力负荷）各有数据或占位 |
| 1.5 | 点击 "手动刷新" 按钮 | 页面重新拉取数据，显示刷新状态 |

### 检查点

- [ ] 无白屏
- [ ] 无控制台错误（F12 → Console）
- [ ] 加载态骨架正常显示

---

## 2. Data Center

| # | 步骤 | 预期结果 |
|---|------|----------|
| 2.1 | 点击底部导航 "数据" 或访问 `/data-center` | 进入数据中心页面 |
| 2.2 | 切换 Tab（睡眠/HRV/静息心率/活动/血氧/压力） | 图表随 Tab 切换更新 |
| 2.3 | 切换时间范围（7天/14天/30天） | 图表数据范围相应变化 |
| 2.4 | 点击 "AI 解读" 按钮 | 弹出 AI 解读模态框，显示图表分析摘要 |
| 2.5 | 观察右侧状态面板 | 显示 "已连接"（绿色）或 "连接异常"（红色） |

### 检查点

- [ ] 图表渲染无报错
- [ ] Tab / 时间范围切换流畅
- [ ] 空数据时显示 "暂无数据"

---

## 3. AI Advisor（智能顾问）

| # | 步骤 | 预期结果 |
|---|------|----------|
| 3.1 | 点击右下角浮动按钮 | Advisor 抽屉从右侧滑出 |
| 3.2 | 在输入框输入 "我最近睡眠怎么样" | 输入框可正常输入 |
| 3.3 | 点击发送或按 Enter | 显示加载动画（三个跳动圆点） |
| 3.4 | 等待回复 | AI 回复以聊天气泡形式显示 |
| 3.5 | 点击智能推荐标签（如有） | 自动填入问题并发送 |
| 3.6 | 关闭抽屉再重新打开 | 历史对话保留 |

### 检查点

- [ ] 发送成功有回复
- [ ] 超时 6 秒后显示 "仔细分析中" 提示
- [ ] 发送失败显示红色系统消息 "发送失败: ..."

---

## 4. God-Mode（上帝模式）

| # | 步骤 | 预期结果 |
|---|------|----------|
| 4.1 | 连续点击页面底部 "AI Health Advisor" 文字 5 次 | 右侧弹出 God-Mode 面板 |
| 4.2 | 在 "切换档案" 下拉中选择不同档案 | 首页/数据中心数据随之切换 |
| 4.3 | 点击 "注入事件" → 选择一个事件 | 页面顶部出现 Active Sensing Banner |
| 4.4 | 点击 "重置全部" 按钮 | 所有覆盖数据被清除，恢复默认 |
| 4.5 | 关闭 God-Mode 面板 | 面板关闭，页面正常显示 |

### 检查点

- [ ] 档案切换后数据更新
- [ ] 事件注入后 Banner 显示
- [ ] 重置后恢复原始状态

---

## 5. Fallback 模式验证

| # | 步骤 | 预期结果 |
|---|------|----------|
| 5.1 | 设置 `FALLBACK_ONLY_MODE=true` 重启 agent-api | 服务正常启动 |
| 5.2 | 刷新首页 | 晨报显示 fallback 内容，标题区域显示 "运行在离线受限模式" |
| 5.3 | 打开 AI Advisor 发送消息 | 回复标记为 "Fallback"（黄色标签） |
| 5.4 | 访问数据中心 | 图表数据正常（来自沙箱数据），AI 解读使用 fallback |

### 检查点

- [ ] 不依赖 LLM API Key 即可运行
- [ ] 所有功能有 fallback 兜底
- [ ] 用户可感知当前为离线模式

---

## 6. 错误处理验证

| # | 步骤 | 预期结果 |
|---|------|----------|
| 6.1 | 停止 agent-api，刷新首页 | 错误提示出现（非白屏），有重试按钮 |
| 6.2 | 访问不存在的路径 `/abc` | 显示 404 页面和 "返回首页" 按钮 |
| 6.3 | 重新启动 agent-api，点击重试 | 页面恢复正常 |

### 检查点

- [ ] 无白屏
- [ ] 有明确的错误信息
- [ ] 重试可以恢复
